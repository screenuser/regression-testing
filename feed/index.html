Freedman https://screenster.io/regression-testing/#comment-769 Thu, 10 May 2018 19:57:39 +0000 http://agileengine.com/?p=1017#comment-769 Regression testing is a perfect example of where automation is an ABSOLUTE must, and UI testing is a great example of a field where writing tests manually isn’t necessarily the best option. It’s really not just that programming tests is difficult or tedious (I’d argue that manual testing is more tedious). WebDriver was created in the era of ‘lean GUI’ where most web UIs were static pages with interaction limited to forms and buttons. For UIs of this type, you didn’t really need to thoroughly test if everything is okay with the layout because all the conversion-critical element of a user journey was concentrated around forms, buttons, and navigation. So basically it was okay to have a script opening a page, checking if the form is there, and if providing the right input values to the form and clicking on submit would give you the right action. Today, you can’t really call this a UI tests. Today things are mostly different on the UI side because we have a lot more logic on the front end. This either requires more manual testing, or a smarter automation solution.
